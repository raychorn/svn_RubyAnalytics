Key Performance Indicators (KPI) Report

The KPI Report allows you to select specific data elements to track. Each of these data elements is can have a target value. If the value of a data elements raises above or falls below you target value, you can be notified of the event. The KPI report appears on the Reports screen in the My Reports category. The KPI report is also one of the reports that appears on the Dashboard.

Type

Mixed – tabular with sparkline

Parameters

●  Sort by Friendly Name ASC (no other sorting)

●  A list of data elements to track. For each data element, collect:

●  Data parameter (optional – connection type, device, error code)

●  Friendly Name

●  Target value

●  Desired value direction – e.g. “Higher is better”, “Lower is better”

●  Alert settings

Data

●  Friendly Name – Use data element name if friendly name not provided.

●  Indicator if value is on the “wrong” side of the target value (“wrong” determined by desired value direction)

●  Current Value

●  Target Value

●  3 Month Trend of value – Chart values for each day (or month for data elements marked with (M)). Mark high and low points and target value.

●  3 Month Change – calculate change from 3 months ago to current time, show “improvement” in green (determined by desired value direction) and “worsening” in red.

●  1 Year Trend of value – Chart values for each day (or month for data elements marked with (M)). Mark high and low points and target value.

●  1 Year Change – calculate change from 1 year ago to current time, show “improvement” in green (determined by desired value direction) and “worsening” in red.

Data Elements

The user may select from the following list of data elements. The “Technology” data elements require the user select a connection type as a parameter. The “Device” data elements require the user select a device as a parameter. The “Error Code” data elements require the user select an error code as a parameter.

Data elements marked with an (M) show monthly values. The trend sparklines should chart monthly values rather than daily values.
